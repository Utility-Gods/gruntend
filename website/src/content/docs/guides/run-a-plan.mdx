---
title: Run a plan through app handlers
description: Keep execution authority in application-owned runtime closures.
---

`defineTools()` is contract-only. Your application supplies the runtime handlers that execute a plan.

```ts
import { createGruntendClient } from "gruntend/client";

const gruntend = createGruntendClient({ tools });

const result = await gruntend.runCodePlan(plan.code, {
  input: plan.input,
  handlers: {
    "menus.list": async ({ ok }) => {
      const menus = await menuRepository.list();
      return ok({ menus });
    },
  },
});
```

Handlers are where your application continues to enforce its rules:

- authorization and tenant boundaries
- validation and invariants
- database transactions
- audit events
- expected errors with `err(...)`

## Treat the plan as untrusted orchestration

A plan can decide *which registered capability to call* and how to combine its results. It cannot reach around the handler layer.

That lets an application support adaptive workflows without handing an LLM direct database or network access.

```ts
const result = await gruntend.runCodePlan(plan.code, {
  input: plan.input,
  handlers,
  onEvent(event) {
    audit.record(event);
  },
});
```

Use one `onEvent(event)` stream for execution lifecycle visibility. The app can render it, log it, or store it as an audit record.
