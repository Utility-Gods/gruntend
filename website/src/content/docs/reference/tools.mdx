title: Tools and handlers
description: The contract and runtime halves of a Gruntend capability.

## `defineTools()`

`defineTools()` declares what an intelligent system may reason about and call.

```ts
const tools = defineTools({
  menu: {
    item: {
      update: {
        description: "Update one menu item.",
        input: updateItemSchema,
        output: updatedItemSchema,
      },
    },
  },
});
```

It does not execute anything. Keep it contract-only.

## `ToolHandlerMap`

A handler map is supplied by the app at runtime. Each handler receives validated input plus `ok(...)` and `err(...)` result constructors.

```ts
const handlers: ToolHandlerMap<typeof tools> = {
  "menu.item.update": async ({ input, ok, err }) => {
    const item = await updateMenuItem(input);
    return item
      ? ok({ item })
      : err({
          code: "ITEM_NOT_FOUND",
          message: "The menu item no longer exists.",
          retryable: false,
        });
  },
};
```

Expected failures should use `err(...)`. Throws remain a fallback for unexpected faults.

## Nested and flat access

A plan can access a dotted tool name through its nested namespace:

```ts
tools.menu.item.update({ menuId, itemId, price });
```

The runtime also supports matching bracket aliases for generated code when needed:

```ts
tools["menu.item.update"]({ menuId, itemId, price });
```
