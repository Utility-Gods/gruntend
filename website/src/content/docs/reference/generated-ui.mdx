---
title: Generated UI
description: Lower-level primitives and framework adapters for mounted generated UI.
---

The generated UI layer is small by design.

## `html`

`createHtmlTag()` creates the tagged-template primitive available to a plan at runtime.

```ts
import { createHtmlTag } from "gruntend/ui";

const html = createHtmlTag();
```

Templates support text, safe attribute values, nested templates, arrays of templates, and function interpolation in approved event attributes.

```ts
html`<button onclick=${save}>Save</button>`;
```

The compiler rewrites event functions to delegated `data-gr-*` attributes. It rejects unsafe tags, unsafe attributes, structural interpolation, and static inline event JavaScript.

## DOM mounting

The framework-neutral DOM primitive mounts a generated component into a DOM element.

```ts
import { mountGeneratedUi } from "gruntend/ui/dom";

const handle = mountGeneratedUi(element, ui, {
  onError(error) {
    console.error(error);
  },
});
```

## Framework adapters

Use the adapter that matches your application:

```text
gruntend/ui/svelte
gruntend/ui/react
gruntend/ui/vue
gruntend/ui/solid
```

Each adapter delegates to the same `mountGeneratedUi()` primitive. The adapters are intentionally thin; the runtime behavior is shared.
